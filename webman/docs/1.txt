🛰️ Pega-Quant 2.0 系统逻辑开发规范
一、 多维趋势识别系统 (Trend Intelligence)
基于 $Boll(400, 2)，通过价格与轨道的相对位置定义三级趋势：
级别        包含周期    核心用途                        识别标准
长期趋势    4H, 1H      决定“主手”方向（做多/做空/观望）上涨：贴近上轨；下跌：贴近下轨；震荡：在中轨附近。
中期趋势    30M, 15M    寻找结构体形成的土壤            同上
短期趋势    5M, 1M      捕捉精确的入场点和V反           同上
趋势连续性追踪 (Historical Sequence)
模型必须记录每个周期的前两个趋势状态，构建“趋势链”：
趋势延续型：下跌 -> 震荡 -> 下跌（空头蓄势，禁止左侧接多）。
趋势反转型：下跌 -> 震荡 -> 上涨（筑底成功，回踩即买）。
V型反转：下跌 -> 上涨（暴力洗盘，小级别极速切入）。
二、 震荡结构体 (X-Y-Z 锚点算法)这是系统捕捉利润的核心。在数据库中以 OscillationStructure 表存储。
1. 关键点定义X点 (Lower Anchor)：下轨外的关键点（$X_1, X_2...$）。若 $X_{new} < X_1$，更新 $X_1$ 为最新极值。Y点 (Upper Anchor)：上轨外的关键点（$Y_1, Y_2...$）。若 $Y_{new} > Y_1$，更新 $Y_1$ 为最新极值。Z点 (Pivot/Midpoint)：$X_n$ 与 $X_{n+1}$ 间的最高价，或 $Y_n$ 与 $Y_{n+1}$ 间的最低价（即回抽/回调位置）。
2. 确认规则 (Confirmation)X1 / Y1 确认：关键点出现后，价格回内轨并触及中轨幅度的 70%。后续点 (Xn / Yn) 确认：价格回内轨并触及中轨幅度的 50%。
3. 结构失效 (Breaker)判定：价格突破 $X_1$ 或 $Y_1$ 的 10%。动作：关闭当前结构体，清空缓存，开启新的空结构体寻找 $X_1, Y_1$。
三、 跨周期交易计划 (Execution Strategy)
1. 计划制定逻辑初始计划：当 $X_1$ 或 $Y_1$ 确认后，自动设定目标区间为 $X_1/Y_1 \pm 8\%$。后续计划：当 $X_n, Y_n$ 多个点位出现后，以其平均价格 $\pm 8\%$ 作为网格挂单区。
2. 入场方式 (Two-Way Entry)静态挂单：在目标区间内，根据用户设定的网格密度（Grid）自动提交限价单（Limit Order）。动态确认 (Confirm-then-In)：当 4H/1H 大级别触及目标区间时，不直接下单，而是等待 1M/5M 小级别形成同样的“震荡结构”或“V反”信号后，市价单立即切入。
3. 仓位管理短期单 (Scalper)：捕捉震荡波动，触碰对侧 Y/X 点或中轨目标平仓。长期单 (Trend-Follower)：利用震荡点位建仓，若结构演变为趋势（向上/向下），则持仓不动，设置保护性止损。
